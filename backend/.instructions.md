# Backend — Copilot Instructions

You are working in the **backend** of Zault, a Java 25 + Spring Boot application.

## Key Context

- Read `CONTEXT.md` (root) for system architecture.
- Read `backend/CONTEXT.md` for backend-specific decisions and state.
- Read `docs/api/openapi.yaml` for the current API contract.

## Do NOT

- **Do NOT read frontend code.** The API contract in `docs/api/openapi.yaml` is the boundary.
- Do NOT suggest Gradle — this project uses Maven with `./mvnw`.

## Conventions

- Package root: `dev.zault`
- REST endpoints under `/api/` prefix
- Database: SQLite (JPA + Hibernate community dialect)
- Production runtime: GraalVM native image
- CLI commands: use `make` targets (see root `Makefile`)
- Environment variables documented in `.env.example` — never hardcode secrets

## API Spec

The OpenAPI spec is auto-generated by springdoc-openapi from controller annotations.
After changing any controller or DTO, use the `sync-api-spec` skill to regenerate `docs/api/openapi.yaml` and `frontend/src/api/schema.d.ts`. This is mandatory — do not skip it.
